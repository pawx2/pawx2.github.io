PA-实现四则运算

最近在做学校的PA时遇到了需要实现四则运算的地方，看讲义也了解到了BNF范式(巴科斯范式)以及eval的递归求值过程。尽管之前在上SICP课程时制作scheme解释器时也用到过eval的框架，不过那个是python语言，加上年代久远且当时仅仅只是使用了而并没有深入理解，因而再次看到时就和第一次见一般，直接感慨BNF范式的精妙。

这里主要记录一下当前所掌握的两种实现四则运算的方法：
1.利用栈这一数据结构将中缀表达式转换为逆波兰式，而后再利用栈计算逆波兰式
2.利用递归加上BNF范式结合eval求值函数对四则运算进行实现

栈-逆波兰式实现

我们已知表达式可以分为三类：前缀表达式、中缀表达式、后缀表达式，而实际上利用栈来实现四则运算，对于前缀表达式和中缀表达式我们可以直接使用两个栈来实现，因为其中涉及括号等对优先级的改变。
不过，在利用后缀表达式求值时，我们则可以只用一个栈，因为运算优先级已经被包含在了运算符的顺序当中。

在“栈-逆波兰式实现”中，我们主要做两件事情：
1.将输入进来的中缀表达式转换为后缀表达式
2.利用栈计算后缀表达式得到结果

中缀表达式转换为后缀表达式

这里首先给出一张表：

符号 # （ *，/，% +，- ）
isp 0 1 5 3 6
icp 0 6 4 2 1

这里的isp表示符号在栈中的优先级(in stack priority)，而icp则表示在栈外，即将进入栈中符号的优先级(in coming priority)。
之后的算法实现步骤就很程式化了：
1.首先将结束符#压入栈中；
2.逐个扫描表达式expr字符获得ch，同时观察栈顶top元素的优先级，根据不同条件执行如下操作：
若ch为#，则遇到结束符，停止该算法
①若ch为数字，则直接将ch压入栈中，ch扫描下一个字符；
②若icp(ch) > isp(top)，则将ch压入栈中，ch扫描下一个字符；
③若icp(ch) < isp(top)，则将栈顶元素退出，并且输出；
④若icp(ch) == isp(top)，则将栈顶元素退出，但是不输出，ch扫描下一个字符；
3.最终输出的字符串就是中缀表达式转换为逆波兰表达式的结果了。

这里给出进一步的解释：
1.括号 ( 的icp优先级最高，是因为只要是括号那么就会优先进行运算，不管前方优先级如何，直接将其压入即可
2.括号 ) 的icp优先级最低，是因为我们首先要计算的是括号里的表达式的值，利用其优先级最低，将直至 ( 的所有符号及数字弹出，转化为逆波兰表达式优先计算
3.( 的isp为1，而 ) 的icp为1，其目的在于运用④若icp(ch) == isp(top)，将栈顶元素弹出，但是不输出，ch扫描下一个字符。可以观察到仅仅只有isp(() == icp())，这里就是为了将两个括号抵消删除。逆波兰式中是不包含括号的，其括号所改变的运算优先级已经蕴含在了数字和运算符的顺序当中，这也是为什么逆波兰表达式仅仅使用一个栈就可以计算的原因。

后缀表达式的运算

在这里，我们已经得到了逆波兰表达式，此时我们仍旧是进行下述程式化步骤，从左至右扫描该表达式获得字符ch，即可计算出结果：
1.若ch为数字，则压入栈中，扫描下一个字符；
2.若ch为符号，则将栈顶的头两个元素弹出，并且利用该ch符号与这两个元素进行运算，所获得的结果压入栈中；

而最后栈中仅剩的最后一个数字即为计算结果。至此，利用栈-逆波兰式对表达式进行计算的过程就结束了。

eval函数框架

这里的eval(p, q)所代表的意义为：求取从tokens数组里从下标p到下标q这一段的数值。
而其中引入的check_parentheses()函数则用于检查对应表达式是否以 ( 开头，以 ) 结尾(即包含在一个大括号内)并且括号匹配，若合乎规范则返回true，否则返回false。
而最后的else一段代码尤其需要注意，我们需要找到的op为dominant operator的下标，也就是优先级最低且不在括号中的运算符在tokens数组里的下标，这里一般为二元运算符，而后将该运算符前、后分别再递归调用到eval函数当中，进行求值。而运算符优先级最低的含义为：1.不同运算符间的优先级最低；2.相同运算符间最右边的运算符优先级最低。
而之所以这里以优先级最低的运算符作为分割，是因为递归是一个自顶向下再自下向上的过程，也就是最先被确定到的op运算符是最后被使用到运算中的，故而要使用最低优先级的运算符作为op，而dominant operator要求在括号之外，则是由于我们还需要对括号表达式进行一个检测和去括号的过程。

另外，我们不难发现，这里的eval函数仅仅只是支持二元简单运算，而对于取负操作-、指针解引用*、取反操作！等一元运算符并没有给予支持，这里我们还可以引入expr函数，在外部，将取负操作与减法操作区分，将指针解引用与乘法操作区分等，而后由于其运算级相对而言是可以归为最高的一类，那么可以判定这类一元运算符仅有其后的表达式而前方是没有其它字符的，从而可以对eval函数进行一些更改。

一些细节

而另外在这个实现过程当中也有一些其它的巧妙的细节：
1.函数传入一个bool类型指针，可以通过这个bool类型在函数返回类型之外另外修改这个bool值，从而以该bool值指示返回值是否有效(等价于多增加了返回值)
2.利用正则表达式来识别token并且将其存入tokens数组，以对数组元素来进行扫描操作以求值。该过程实际上也是众多编译器翻译识别源代码的基本原理。
