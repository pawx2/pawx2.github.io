题目为Leetcode.142环形链表II

给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。
如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。
不允许修改 链表。

解题步骤

Step1.使用快慢指针，快指针一次走两步，慢指针一次走一步；
Step2.两个指针第一次相遇，将慢指针指向头节点；
Step3.快慢指针均一次走一步，第二次相遇则指向环的入口节点；

原理解析

整个原理如上图所示。
graph-1，起初快指针fastp和慢指针slowp均位于头节点；
graph-2，当慢指针到达了环的入口时，由于慢指针走一步，快指针走两步，则快指针走过的总距离为2x，在环内走的距离为x。而两个指针顺时针走，快指针想要追上慢指针则需要额外走 c(环的周长) - x 距离，这也意味着慢指针要走 c-x 的距离。
graph-3，慢指针走了c-x距离，此时快慢指针第一次相遇，将慢指针指向头节点，并且快指针此后每次只走一步。可以看到，快慢指针到达环入口的距离均为x，因此，当快慢指针第二次相遇时即为环入口节点。
另外，这里为了方便仅用了 c >= x 的情况来进行解释，当 c < x 时，则可以用模进行推广(有环可以容易想到模系统)，同样可以验证其正确性。
