前言

纸牌博弈问题
排成一排的纸牌，每张纸牌都有其点数，博弈双方轮流每次从左端或右端不放回地抽取纸牌，假定博弈双方都足够聪明，求最终先手方和后手方分别能够获得的点数。

该问题为经典的DP动态规划问题，而根据限制条件可以分为两个问题：
1.每次只能抽取一张纸牌，基础版本；
2.每次可以抽取连续多张纸牌，进阶版本；

单张抽取

问题描述

可见：Leetcode.486 预测赢家

Description：
给定一个整数数组，双方轮流每次从整数数组左端或右端每次不放回地抽取一个数字，假定双方都足够聪明，最后依照双方各自所得数字之和判定胜负：
先手方获得数字之和 >= 后手方获得数字之和，先手方获胜；
否则，后手方获胜；
Input：整数数组Cards[a1, a2, a3, a4, …, an]
Output: 先手方能赢得游戏 True，否则 False；

解决思路

解决该问题很重要的两个要点在于：
1.解题思路基于区间扩增，而不是随时间扩增；
2.双方博弈对抗实际上可以视为自己和自己对抗；

首先是第一个要点：
看到问题中的“轮流”，我们很容易陷入一个误区，即按照时间线/对局的进行来考虑问题。但实际上认识到其作为动态规划问题后，我们不难发现大问题的解决依赖于小问题，动态规划实质为有组织的Brute Force（蛮力求解），以优化时间复杂度。而要获得所给“大数组”情况下的问题结果，我们从“小数组”情况开始考虑，进而不断拓展问题规模，最终解决问题。

其次是第二个要点：
博弈双方的引入实际上也可能会困扰我们的思路，但倘若我们换个角度，自己和自己对抗，则问题就变为了“最优选择”与“最差选择”的交替进行，从而进行动态规划。

连续抽取

问题描述

Description:
给定一个整数，双方轮流每次从整数数组左端或右端每次不放回地连续抽取数字（每次至少抽取一个数字），假定双方都足够聪明，最后依照双方各自所得数字之和判定胜负：
先手方获得数字之和 >= 后手方获得数字之和，先手方获胜；
否则，后手方获胜；
Input：整数数组Cards[a1, a2, a3, a4, …, an]
Output: 先手方能赢得游戏 True，否则 False；

解决思路

大致思路同每次只允许抽一张的情况一致，我们从Brute Force蛮力策略O(n^3)开始考虑，而后通过DP优化将时间复杂度降低至O(n^2)。

Brute Force蛮力策略

即直接暴力求解，以Cards[2, 7]为例，sums[i, j]表示牌组Cards从i -> j范围内所有牌的点数之和：

那么一个很显然的想法是：基于先前每次仅抽取一张牌的算法，在运行前先用O(n^2)时间复杂度计算出sums数组，而后每次对 f[i, j] 和 s[i, j] 更新时都使用另一个变量k，k: i -> j来遍历所有情况进行更新，这样形成一个三层 for循环 嵌套，不难验证时间复杂度为O(n^3)。

DP优化

通过观察上述状态转移实例：

我们不难发现：

  1. 对于 s[i][j] 的状态转移，实际上仅仅只依赖于当前已得出的 f 第 i 行中的最小值与当前已得出的 f 第 j 列中的最小值。那么显然的一个思路为：

    利用两个一维矩阵 slmin、srmin 分别记录 f 各行的最小值、f各列的最小值，而后更新 s[i][j] 时仅需要s[i][j] = min(slmin[i], srmin[j]);即可，而对 slmin 和 srmin 的更新则可以在每次更新 f 后进行更新，因为依赖于先前分析的更新顺序，在更新到 s[i][j] 时，其所需的 f 元素早已计算完成。

  2. 对于 f[i][j] 的状态转移则没有那么明显，如果我们正面思考，则会陷入困局：固定左侧 l，右侧 r，中间以 k 遍历获取最差状态，则总是要三层 for循环 从而无法优化时间复杂度。
    但是如果我们反向思考：我方先手获取到最优 == 对方获取到最差，而又已知 sums[i][j] 实际上对于 i -> j 范围内的讨论其总为定值，那么有如下分析；

    而观察上述，我们同样可以知道，对方所能获得的最差同样仅与固定行、固定列有关，从而我们可以构造两个一维矩阵 flmin、frmin 来记录固定行、固定列下对手所能获得的最小值, 仿照对 s[i][j] 的更新来完成对 f[i][j] 的更新：f[i][j] = max(sums[i][j] - flmin[i], sums[i][j] - frmin[j]);。
