LEC-ADA-EX-Chapter21 'NP完全性理论初步'习题解答

Posted on 2022-06-06 In LEC-ADA-EX
《算法设计与分析》黄宇 第二版 Chapter21 'NP完全性理论初步'习题解答，来自NJU算法设计与分析课程(LADA, Lectures of Algorithm Design And Analysis)

已识乾坤大，犹怜草木青 。
长空送鸟印，留幻与人灵。
马一浮

21.1

  • 某个NP完全问题为NP-Hard问题，则所有NP问题都可以多项式时间归约到该问题解决，
    而若该问题可在O(poly(n))解决，
    则所有NP问题都可以先多项式时间归约到该问题，再解决该问题，
    即所有NP问题都可以在多项式时间内解决，从而P=NP；
  • NP完全问题Q不存在多项式时间解，则Q也属于NP问题，则对于所有NP-C问题，由于为NP-Hard问题，则对于任意一个NP-C问题X，Q可以多项式时间归约到X进行解决，而Q又不存在多项式时间解，则X不存在多项式时间解，进而所有的NP完全问题都不可能在多项式时间解决；

21.3

  • 从n个点中选取出k个点，其时间复杂度为O(n!/((n-k)!k!)) 近似等于O(n^k)，而检验这k个点是否能构成团则时间复杂度为O(k^2)。从而可设计算法：

  • 未证明，CLIQUE算法会由k：1->n 地调用该算法，从而有O(n) + O(n^2) + … + O(n^n) != O(poly(n))，这里的k不再是定值，而是一个变量

21.4

  • 假定输入DNF范式由m个子句组成，每个子句里有n个条件变量，则不难构造O(poly(n))的Alg算法解决问题：
    遍历m个子句，如果在子句中同时出现了 a 和 !a ，那么删除该子句，最后考察剩余子句数量，如果为空返回False，否则返回True。
    时间复杂度为O(mn) = O(poly(n))，可在多项式时间下解决，因而属于P问题
  • 错误在于：未严格限制归约为多项式归约，CNF范式转为DNF范式的时间复杂度就超过了O(poly(n))，从而归约引入了非多项式因素，进而未严格说明CNF-SET问题能够在O(poly(n))内解决，从而也就无法验证其为P问题，更没有P=NP。

21.5

    1. 构造从 子集和问题 => 背包问题 的归约，将传入集合作为物品重量、价值传入，而目标和S作为容量、价值下界传入。并且从形式、意义、多项式时间上说明该归约的建立；
    2. 依据NP-Hard问题定义、多项式归约传递性，证明背包问题为NP-Hard问题：
      由 子集和问题为NP-C问题 => 子集和问题为NP-Hard问题 => 子集和问题可以归约到 背包问题解决 => 背包问题为NP-Hard问题
    3. 证明背包问题为NP问题；
    4. 依据NP-C及上述推理，说明背包问题为NP-C问题

  • 可构建如下DP描述及状态转移方程

  • DP算法时间复杂度为O(Cn), C为背包容量，以二进制形式考虑C，若表示C的二进制串长度为L，则时间复杂度为O(2^Ln)，属于伪多项式时间算法

21.6到21.10都为NP-C问题的判定，解题流程可有如下模板：
三步走：
①已知NP-C 归约到 待求问题（形式、意义、多项式时间）；
②证明待求问题为NP问题；
③据NP-C定义，说明待求问题为NP-C问题

21.6

将点视为集合，与点直接关联的边视为集合当中的元素，从而归约到最小集合覆盖问题解决。

21.7

构造一个完全图，而后将完全图作为母图，从图G中选定一点，向外围逐点拓展形成图S，每次拓展完后将图S作为子图，而后母图、子图传入SUBGRAPH-ISOMORPHISH的Alg中进行解决，如果返回False，则将刚刚加入的点删除，否则保留下来，最后形成团为最大团。调用n次，为多项式时间归约。

21.8

构造独立集问题->强独立集问题的多项式归约：
将G中有连接的点，全部变为一个个点对，如果出现了多余的点则直接删除。而后将新图输入到独立集问题Alg当中，返回的k即为问题解。

21.9

构造哈密顿回路问题->0权环问题的多项式归约：
对于输入图G有n个顶点，选定一个源点，而后遍历其直接关联的边，每次选择一条边，将其权设置为-(n-1)，而后其它边权均设置为1，之后输入0权环问题Alg进行求解。

21.10

构造有向哈密顿路径问题->EPP问题的多项式归约：
传入EPP问题中的区域集合为“每一个点就是一个区域”，而后可解决有向哈密顿路径问题
