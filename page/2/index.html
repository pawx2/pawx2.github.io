求最大公因数的三种方法

前置知识🧀

若学习过数论/《离散数学》课程，可以很容易理解算法的正确性证明，若未学习过，下面列出部分说明帮助理解：

a | b，表示“a可以整除b”
gcd(a, b)，表示“数字a、b的最大公因数”
任意两数a、b可以相互表示 a == kb + c，简单线性函数构造

辗转相除法

正确性证明：

证明：
设有数字a，b且b > a，则可以表示a、b间关系为 b = ka + c (k、c为常数)
并且假设有 d | a 并且 d | b，则因为 c == b - ka 易推出 d | c (即 d | (b - ka))
而这里 d 可以为任何 a、b 共有因数的集合，则也包含了两者的最大公因数
因而有 gcd(a, b) = gcd(a, b % c)，并且当 a == b 时可以得到最大公因数

更相减损术

正确性证明：

证明：
设有数字a，b并且b > a
并且假设有 d | a 并且 d | b，那么显然可以推出 d | (b - a)
而这里的 d 可以表示 a、b两数的任意共有因子，也包含了两者的最大公因数
从而我们可以推出 a 与 b - a 拥有相同的因子集合
并且当 a == b 时，此时的 a 即为最大公因数

Stein算法(辗转相除法+更相减损术+位移运算)

之前还真不知道有这么个算法，开眼界了属于是🤡

正确性证明：

其正确性依赖于 辗转相除法 与 更相减损术 此处不再证明

算法原理：

移位运算性能快，并且结合以下规律可以提出Stein算法
1.a、b均为偶数，则有gcd(a, b) == 2 * gcd(a/2, b/2) == 2 * gcd(a >> 1, b >> 1);
2.a为偶数，b为奇数，则有gcd(a, b) == gcd(a/2, b) == gcd(a >> 1, b);
3.a为奇数，b为偶数，则有gcd(a, b) == gcd(a, b/2) == gcd(a, b >> 1);
4.当a、b均为奇数时，利用更相减损术，则有gcd(a, b) == gcd(b, a-b)，a-b必然为偶数，又转化为上述情况

OJ-滑动窗口丨双指针-Leetcode.76最小覆盖子串

这题确实是算一道经典的Hard题了，磨了我小几个小时也没A出来，寄！

题目为Leetcde.76最小覆盖子串

给你一个字符串 s 、一个字符串 t 。返回 s 中涵盖 t 所有字符的最小子串。如果 s 中不存在涵盖 t 所有字符的子串，则返回空字符串 “” 。

注意：
对于 t 中重复字符，我们寻找的子字符串中该字符数量必须不少于 t 中该字符数量。
如果 s 中存在这样的子串，我们保证它是唯一的答案。

示例 1：
输入：s = “ADOBECODEBANC”, t = “ABC”
输出：”BANC”

解题步骤

Step1.扫描一遍字符串t，利用两个一维数组记录字母是否出现、出现次数；
Step2.双指针从字符串s开始，右指针移动，拓大滑动窗口以包含t中所有字母，另外，修改两个记录数组，适时调整左指针以缩小滑动窗口满足题意中“最小覆盖”条件。
即：右指针扩大，当字符串t被全部包含时，左指针缩小以尝试获得最优解。

解题原理

优先向右移动右指针扩大范围，包含字符串t中所有字符；
在字符串t中所有字符被包含前提下，尝试向右移动左指针，缩小s的子串长度，并记录下最小长度及字串起始位置。

解题步骤

Step1.使用快慢指针，快指针一次走两步，慢指针一次走一步；
Step2.两个指针第一次相遇，将慢指针指向头节点；

原理解析

整个原理如上图所示。
graph-1，起初快指针fastp和慢指针slowp均位于头节点；
graph-2，当慢指针到达了环的入口时，由于慢指针走一步，快指针走两步，则快指针走过的总距离为2x，在环内走的距离为x。而两个指针顺时针走，快指针想要追上慢指针则需要额外走 c(环的周长) - x 距离，这也意味着慢指针要走 c-x 的距离。
graph-3，慢指针走了c-x距离，此时快慢指针第一次相遇，将慢指针指向头节点，并且快指针此后每次只走一步。可以看到，快慢指针到达环入口的距离均为x，因此，当快慢指针第二次相遇时即为环入口节点。
另外，这里为了方便仅用了 c >= x 的情况来进行解释，当 c < x 时，则可以用模进行推广(有环可以容易想到模系统)，同样可以验证其正确性。

题目为Leetcode.1001网格照明

在大小为 n x n 的网格 grid 上，每个单元格都有一盏灯，最初灯都处于 关闭 状态。
给你一个由灯的位置组成的二维数组 lamps ，其中 lamps[i] = [rowi, coli] 表示 打开 位于 grid[rowi][coli] 的灯。即便同一盏灯可能在 lamps 中多次列出，不会影响这盏灯处于 打开 状态。
当一盏灯处于打开状态，它将会照亮 自身所在单元格 以及同一 行 、同一 列 和两条 对角线 上的 所有其他单元格 。
另给你一个二维数组 queries ，其中 queries[j] = [rowj, colj] 。对于第 j 个查询，如果单元格 [rowj, colj] 是被照亮的，则查询结果为 1 ，否则为 0 。在第 j 次查询之后 [按照查询的顺序] ，关闭 位于单元格 grid[rowj][colj] 上及相邻 8 个方向上（与单元格 grid[rowi][coli] 共享角或边）的任何灯。
返回一个整数数组 ans 作为答案， ans[j] 应等于第 j 次查询 queries[j] 的结果，1 表示照亮，0 表示未照亮。

解题原理

直接暴力求解必然导致超时，这里采取线性映射配合哈希表进行解决。
原理图如下：

Step1.使用四个哈希表映射每个行、列、正对角线(左上->右下)、反对角线(右上->左下)中亮灯数量

映射关系为：
行、列直接映射；
对角线通过截距映射(以查询点为中心画X，两轴截距唯一)

Step2.查询点时如果对应行列及对角线上亮灯数>0，则该处照亮，并熄灭以该点为中心九宫格的九个区域的灯
Step3.返回结果

注意点

1.注意使用亮灯对四个哈希表初始化时过滤重复（同一地点亮灯多次），这里需要使用到集合
2.注意熄灯指的是熄灭有亮灯的区域的灯，并且对四个哈希表进行更新；被照亮区域也是亮的，但是熄灯操作对其和哈希表无任何影响
3.熄灯时注意对集合也进行更新，否则导致小bug

DS-判断图是否有环的三种方法

Posted on 2022-02-07 In DS-OJ

DS-判断图是否有环的三种方法

拓扑排序

拓扑排序规范后又称为Kahn算法，是判断图中是否有环的最常用的方法。
拓扑排序对于有向图/无向图的带环判断是基本一样的，可以说是相当统一。

拓扑排序需要使用到如下数据结构

队列，用于存放节点；
邻接表，用于存放每个节点指向的节点；
一维数组，用于记录每个节点的入读；

拓扑排序判断图是否有环可以通过如下几个步骤进行：

Step1.根据所给数据构图，完善邻接表及入度数组
Step2.初始化队列，将入度为零节点追加到队列中
Step3.进入循环，队列为空中止循环，每次循环弹出队首节点，查询邻接表，将其指向所有节点的入度减一，如果节点入度变为0则追加到队列尾部，另外记录弹出节点数量
Step4.对比记录的弹出节点数量与总节点数量，便可判断图中是否有环

DFS深度优先遍历

通过调整深度优先遍历可以用于判断图中是否存在环。
DFS判断图是否带环，有向图/无向图两种情况大致相同。

整个算法的思路主要为如下两点：

Point1.每次算法进行时仅能对一个连通分量进行判断，因此最外层循环遍历所有连通分量套用算法
Point2.每个节点有三种颜色：白，灰，黑。设定初始时节点为白色，当节点被遍历后变成灰色，而当与之直接或间接相连的节点都被遍历后其变为黑色。那么在遍历过程中，如果在遍历了一个节点后遍历其邻接节点，且该节点为灰色并且不为上一个遍历到的节点，则出现了环。

并查集

并查集的经典题型就包含了对图中环的判断，也是一种常用的方法。
并查集对于有向图/无向图的带环判断有细微不同。

算法思路如下：

Step1.通过遍历，将相互间直接/间接相连的节点归属到同一个集合中，并且在遍历过程中不断合并集合。
Step2.对于无向图，当遍历到u->v时，如果u,v拥有相同的父亲，那么说明有环；对于有向图，当遍历到u->v时，如果v是u的父亲，则说明有环。

DS-BFS-带传送门的迷宫

这几天死磕学校的OJ题目，数据结构课程中学队列的时候来了一道经典的BFS问题，虽然说和队列这个数据结构是有关系，但是关系不大，更大的是BFS(广度优先搜索)这个算法。题目呢就是一道经典的迷宫最短路径的变式，添加了传送门，但是实际上还是使用原有的BFS，只不过考察你对精髓掌握的深刻不深刻了。

预备知识

这里我就直接口述了(bushi)，毕竟利用图论术语来说明BFS多少有点催眠(可能只有本菜鸡🐓这么觉得…)。

BFS就是广度优先搜索，可以理解为从一个点 S 开始，将距离该点为1的所有点 Si 存储在一个队列里，之后逐个遍历 Si，对于每个没有走过的 Si ，遍历到它的时候将距离 Si 为1的所有点 Sj 加入到队列尾，将 Si 从队列头出队，并标记 Si 已经走过，等到所有 Si 遍历完后，这个时候队列里就全是 Sj (距离Si距离为1的所有点 or 距离 S 距离为2的所有点)，如此循环下去，直到遇到了某个点为目标终点，那么就返回这个到这个点时的一个特征值(比如为每个点都标记一个属性描述从 S 到该点走了多少步，那么每次从 Si 到 Sj，Sj 的这个特征值就为 Si 的加1)。

其实到这里已经将最经典的迷宫最短路径的求解思路讲完了。不过作一些拓展，学过的离散数学/图论的同学都会知道 Dijkstra 算法，而 BFS 和 Dijkstra 算法并没有什么太大的区别，不过是 BFS 运用于无权图，而 Dijkstra 运用于带权图。

我们这里还可以为 BFS 添加一个更有“感觉”的描述，那就是水的波纹。我们不妨设想一下，平静的水面上飘着一片树叶🍃，以树叶作为我们的“目标终点”，并且水面上有许多障碍物。一滴水滴💧打入水面，水滴的落点就是起点，那么水滴💧落下，波纹扩散下去，一直到波纹波及树叶，这个时候就找到了从起点-水滴💧->树叶🍃的最短路径了。(当然，把路都封死了是波及不到了。还有，别和我说物理，我不听我不听，自己多多感受一下，会有感觉的)

解题思路

实际上上一个部分已经讲的差不多了O(∩_∩)O。

这道题我们来看，最基本的是要找走迷宫的最短路径，那么利用典型的BFS就可以解决了，先造一个队列 Queue，从起点 S 开始，将能走到且距离为1的所有点 Si (即上下左右且不是障碍的点)从队列尾部入队。那么这个时候问题就来了:

如果这个点是个传送门，并且可以任意传送到其它的传送门，那么我该继续用BFS吗，怎么走？

这里实际上就是变式的问题所在了。但是仔细思考，实际上和单纯B的最短路径是没有什么区别的，只是到了传送们的格子时我们可以分为两种情况：1.不进入传送门，只是停留在这个格子里；2.开启传送门，遍历所有的其它为走过的传送门。而由BFS我们知道，在此之前的，比传送到其它传送门的路径都走过了，故只要进入传送门，就会离起点更远，不可能离起点更近，也就是说至少是有机会离终点更近的。那么基于以上分析我们就可以写出代码，这题的变式也不过是添加了一个分支，换一个地方进行BFS。

最后，我们需要注意，当一个点我们将要走到或已经走到了，我们需要对其进行标记，使其之后不能再走，否则就会出现传送门之间疯狂循环或者是走了“回头路”的情况，进而得到不正确的答案。

DS-并查集丨图-吃鱼

最近学校的DS(数据结构课)在上集合有关的内容，前几天也遇到了一题有关于并查集的题目。一开始怎么想都是越想越复杂，甚至是根本没有往并查集方面去思考，最后也是在一点点的提示之下才明确了思路，而后不久利用递归法对其进行了实现，并且在实现后深深的感慨递归自顶向下再自下向上的巧妙，故在此记录一下，同时也重新整理一下思路。

并查集

并查集这方面的内容实际上可以利用图论里的连通图来思考，这里就不进行赘述了，因为涉及离散数学方面的内容。而并查集实际上展开说就是合并与查找集合的一个过程，其目的是将集合中有关系的元素划分为一个组（术语称之为等价类，而提供的关系称之为等价关系），而相互之间毫无关系的元素会分属在不同的组中。

将含有一堆元素的集合最后分为一个个互相毫无关系的组，而同一个组内的任意元素之间相互间接或直接有关系，这就是并查集所做的事情。

我们可以拿一个现实中的事情来举例：

比如有这么一群人，他们都遵循“朋友的朋友是朋友的原则”，那么将这样一群人作为一个群体，我们得到了：
{李华} {张三} {李四} {王五} {明明} {托米} {劳拉} {憨憨}，由于并不知道那些人有关系，所以每个人都各为一个集合。
而后我们不断给出一个“对”（pair），表面两人是朋友关系，比如（小白，小黑）就表示小白和小黑互为朋友，我们将相互为朋友的人放在一个集合，也就是分为同一个组，那么我们可以得到一个如下过程：
step1.
给出pair（李四，张三）
说明李四和张三是朋友
当前有组：{李四， 张三}
剩余人物：{李华} {王五} {明明} {托米} {劳拉} {憨憨}

step2.
给出pair （李四，王五）
说明李四和王五是朋友
当前有组：{李四，张三，王五}
剩余人物：{李华} {明明} {托米} {劳拉} {憨憨}

step3.
给出pair （李华，憨憨）
说明李华和憨憨是朋友
当前有组：{李四， 张三，王五}， {李华， 憨憨}
剩余人物：{明明} {托米} {劳拉}

step4.
给出pair （王五，李华）
说明李华和王五是朋友，而李华有朋友憨憨，那么王五也就是憨憨的朋友，憨憨也就是组1里各个成员的朋友
当前有组：{李四，张三，王五，李华，憨憨}
剩余人物：{明明} {托米} {劳拉}

…之后一直重复上述流程，直至不再给出pair，而将人物换成元素，将“朋友关系”变换成等价关系，那么也就是正式的并查集的流程了。

简单应用

这道题目源于Leetcode上的原题，序号为947。

问题

小蓝鲸最喜欢的食物当然是小鱼了。假设在二维平面的整数坐标点上有一些小鱼，并且每个位置最多只有一条小鱼。如果一条小鱼的同行或者同列上有其他小鱼存在，那么小蓝鲸就可以吃掉这条小鱼。所以小蓝鲸最多可以吃掉多少条小鱼呢？

输入

输入第一行一个整数n，表示小鱼的个数。

接下来n行每行两个空格分开的整数x和y，表示每条小鱼的坐标（输入数据保证每条小鱼的坐标互不相同）。

输出

输出一个整数，表示小蓝鲸最多可以吃掉小鱼的数量。

首先贴出我用递归求解的源代码

首先我们理清楚解决问题的办法：由于只要存在有同行或同列的元素，那么该元素就可以被删除。这一条件说明所有具有相同行、列的元素，无论直接还是间接都可以构成一个组，而后选取其中一个元素作为“根”，由最外层向内不断删除直至遇到根，最后保留根，那么删除的数量就是所能删除的最大数量了。而后我们反向计算得到：能删除的最大数量=总数量-根数量。（实际上这里的思考角度就是离散数学里连通图的思考角度，我们只要统计连通图数量，而后用元素总数减去连通图数量就得到该题的答案了）

这里的思路实际上还是挺清楚的，get_data读入数据并且存入到一个大数组里，这个数组为一个二维数组，一个元素为一个三元素组，存储着（x, y, exist），x就是其x坐标，y就是其y坐标，exist则是当前元素是否存在的指示位。（也就是该位置的鱼🐟是否没有被吃掉）。

而后我们通过在solution函数下利用find_map递归遍历数组，每遍历到首个存在的（exist为1的）元素，我们就将其exist置为0，即将其删除，而后再向后遍历，倘如遍历到了有和当前元素相同行或列的元素，再对该元素递归调用find_map删除与其有关的所有元素。最后我们用总的元素减去map_num所积累下来的元素个数（也就是作为“根”的元素的个数）就是能够被吃掉的小鱼🐟的最大数量了😭

这里递归比较巧妙的地方，大概就在于随着递归进行会在一个线性的数组向后不断修改，从根拓展向外不断删除元素的这么一个过程（精神恍惚🥱，其实就是很经典的一个递归过程）

对这里的指令进行一些解析:

  • -P xxxx 指定命令使用的端口号xxxx（拷贝目标文件至少得先走对路）

在这个过程中也遇到了不少的问题。
一方面，一般远程主机提供的Linux系统要访问对应的端口都是需要账户登录的，不然是没有权限获取文件的，并不像github上的公共目录，直接git clone下来就行了，所以scp命令的执行过程之中是需要正确输入对应的密码等信息的。
另一方面，scp指令一定要选对端口，否则可能scp指令运行到一般会一直卡住。一开始我使用的端口是学校提供的终端网址的IP端口，但实际上这里的端口是网页的端口，并不是终端实体所提供的Linux系统的端口，这样想要从远程拷贝文件到本地就是“牛头不对马嘴了”:smile:
